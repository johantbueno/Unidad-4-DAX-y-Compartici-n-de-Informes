DAX: Data Analysis Expressions

El lenguaje de fórmulas para análisis avanzado en Power BI

¿Qué es DAX?

DAX (Data Analysis Expressions) es un lenguaje de fórmulas desarrollado por Microsoft para Power BI, Power Pivot y Analysis Services.

Permite crear cálculos personalizados desde operaciones simples hasta análisis complejos de inteligencia temporal.

Características principales:
  • Sintaxis similar a Excel (pero más potente)
  • Trabaja con tablas y relaciones
  • Contextos de filtro y fila
  • Más de 250 funciones incorporadas

¿Para qué se usa?

  • Medidas: Cálculos dinámicos que responden a filtros (ej: Total Sales, Profit Margin)
  • Columnas calculadas: Nuevas columnas basadas en otras columnas de la tabla
  • Tablas calculadas: Crear nuevas tablas desde cero o transformando existentes
  • Time Intelligence: Análisis comparativos (YTD, MTD, Year-over-Year)
  • KPIs complejos: Métricas empresariales personalizadas

Fundamentos de DAX

Medidas (Measures) Básico

Cálculos que se evalúan dinámicamente según el contexto de filtro.

DAX
// Total de ventas Total Sales = SUM(Orders[Sales]) // Margen de beneficio Profit Margin = DIVIDE( SUM(Orders[Profit]), SUM(Orders[Sales]) )
Columnas Calculadas Básico

Se calculan una vez y se almacenan en la tabla. Se evalúan fila por fila.

DAX
// Precio unitario Unit Price = Orders[Sales] / Orders[Quantity] // Categoría de venta Sale Category = IF( Orders[Sales] > 1000, "Alto", "Bajo" )

Funciones DAX Esenciales

SUM / SUMX
// Suma simple Total Revenue = SUM(Orders[Sales]) // Suma iterativa (fila por fila) Total Cost = SUMX( Orders, Orders[Quantity] * Orders[Unit Cost] )
AVERAGE / AVERAGEX
// Promedio simple Avg Sales = AVERAGE(Orders[Sales]) // Promedio ponderado Avg Discount = AVERAGEX( Orders, Orders[Discount] )
COUNT / COUNTROWS
// Contar valores no vacíos Product Count = COUNT(Products[ProductID]) // Contar filas de tabla Order Count = COUNTROWS(Orders)
MAX / MIN
// Valor máximo Max Sale = MAX(Orders[Sales]) // Valor mínimo Min Sale = MIN(Orders[Sales])
CALCULATE

La función más importante de DAX. Modifica el contexto de filtro.

// Ventas solo de categoría Furniture Furniture Sales = CALCULATE( SUM(Orders[Sales]), Orders[Category] = "Furniture" ) // Ventas año anterior Sales LY = CALCULATE( [Total Sales], SAMEPERIODLASTYEAR(Calendar[Date]) )
FILTER

Retorna una tabla filtrada que se puede usar en otras funciones.

// Ventas con descuento mayor a 20% High Discount Sales = CALCULATE( SUM(Orders[Sales]), FILTER( Orders, Orders[Discount] > 0.2 ) )
ALL / ALLEXCEPT
// Total sin filtros Total Sales All = CALCULATE( SUM(Orders[Sales]), ALL(Orders) ) // % del total % of Total = DIVIDE( [Total Sales], [Total Sales All] )
RELATED
// Traer datos de tabla relacionada Product Category = RELATED(Products[Category]) // En columna calculada de Orders Customer Segment = RELATED(Customers[Segment])
Requisito: Las funciones de Time Intelligence requieren una tabla de calendario (Date Table) con fechas continuas.
YTD / QTD / MTD

Acumulados hasta la fecha (Year/Quarter/Month To Date)

// Ventas acumuladas del año Sales YTD = TOTALYTD( [Total Sales], Calendar[Date] ) // Acumulado del mes Sales MTD = TOTALMTD( [Total Sales], Calendar[Date] )
Comparaciones Temporales
// Mismo período año anterior Sales LY = CALCULATE( [Total Sales], SAMEPERIODLASTYEAR(Calendar[Date]) ) // Crecimiento Year-over-Year YoY Growth = DIVIDE( [Total Sales] - [Sales LY], [Sales LY] )
DATEADD
// Ventas mes anterior Sales Previous Month = CALCULATE( [Total Sales], DATEADD(Calendar[Date], -1, MONTH) ) // Ventas próximo trimestre Sales Next Quarter = CALCULATE( [Total Sales], DATEADD(Calendar[Date], 1, QUARTER) )
DATESBETWEEN
// Ventas en rango específico Sales in Period = CALCULATE( [Total Sales], DATESBETWEEN( Calendar[Date], DATE(2024,1,1), DATE(2024,6,30) ) )
IF / SWITCH
// IF simple Sales Level = IF( [Total Sales] > 10000, "Alto", "Bajo" ) // SWITCH para múltiples casos Region Name = SWITCH( Orders[Region], "E", "Este", "W", "Oeste", "Unknown" )
AND / OR
// Condiciones múltiples AND Premium Sales = CALCULATE( [Total Sales], AND( Orders[Sales] > 1000, Orders[Profit] > 200 ) ) // Condiciones múltiples OR Special Sales = CALCULATE( [Total Sales], OR( Orders[Category] = "Tech", Orders[Sales] > 5000 ) )
Mejores Prácticas DAX
  • Usa medidas en lugar de columnas calculadas cuando sea posible (mejor performance)
  • Nombra claramente tus medidas (ej: "Total Sales" en lugar de "Measure1")
  • Formatea números adecuadamente (moneda, porcentaje, decimales)
  • Comenta tu código DAX para futuras referencias
  • Evita funciones anidadas complejas - divídelas en medidas intermedias
  • Usa variables (VAR) para cálculos repetidos y mejor legibilidad

Compartir Informes en Power BI

Métodos para distribuir y colaborar en tus dashboards

¿Por qué compartir informes?

Compartir informes de Power BI permite democratizar el acceso a datos y análisis en tu organización, facilitando la toma de decisiones basada en datos a todos los niveles.

Power BI ofrece múltiples métodos de compartición según tus necesidades de colaboración, seguridad y audiencia.

Métodos de Compartición

1. Compartir Enlace Directo

¿Cómo funciona?

  • Genera un enlace único al informe
  • Envía por email o Teams
  • Los usuarios necesitan licencia Power BI

Pasos:

  1. Publica informe en Power BI Service
  2. Clic en "Compartir" en el informe
  3. Ingresa emails de destinatarios
  4. Configura permisos (ver/editar)
  5. Enviar invitación
Ideal para: Colaboración rápida con colegas internos
2. Publicar en Workspace

¿Cómo funciona?

  • Sube informe a espacio de trabajo compartido
  • Gestiona permisos por workspace
  • Colaboración estructurada en equipo

Pasos:

  1. Crea workspace en Power BI Service
  2. Asigna miembros (Admin, Member, Contributor, Viewer)
  3. Publica desde Desktop al workspace
  4. Configura actualización de datos
  5. Usuarios acceden desde workspace compartido
Ideal para: Equipos que trabajan en múltiples informes
3. Publicar en Web (Público)

¿Cómo funciona?

  • Genera código embed HTML
  • Acceso público sin autenticación
  • Incrusta en sitios web o blogs

Pasos:

  1. Abre informe en Power BI Service
  2. Archivo → Insertar → Publicar en web
  3. Confirma riesgos de seguridad
  4. Copia código embed o enlace
  5. Inserta en tu sitio web
¡Cuidado! Solo usa con datos públicos (sin información sensible)

Métodos Adicionales

Power BI Apps

¿Qué es? Empaqueta múltiples informes y dashboards en una app distribuible.

Ventajas:

  • Experiencia simplificada para usuarios finales
  • Actualización centralizada de contenido
  • Permisos a nivel de app completa

Uso: Workspace → Crear app → Configurar navegación → Publicar

Suscripciones por Email

¿Qué es? Envío automático de snapshots del informe por correo.

Ventajas:

  • Usuarios reciben informes periódicamente
  • Configurable por horario (diario/semanal/mensual)
  • No requiere abrir Power BI Service

Uso: Informe → Suscribirse → Configurar frecuencia → Añadir emails

Exportar a PDF/PowerPoint

¿Qué es? Descarga estática del informe en formato PDF o PPTX.

Ventajas:

  • Compartir sin necesidad de Power BI
  • Ideal para presentaciones offline
  • Conserva diseño visual

Limitación: No es interactivo, es una imagen estática

Integración con Teams

¿Qué es? Inserta informes directamente en canales de Microsoft Teams.

Ventajas:

  • Acceso directo desde Teams
  • Colaboración en tiempo real
  • Notificaciones de actualización

Uso: Canal de Teams → + → Power BI → Seleccionar informe

Publicar en Web No Sí Ninguna (Público) Datos públicos en sitios web

Suscripción Email Sí No (snapshot) Media Reportes periódicos automáticos

Seguridad y Control de Acceso

Protege tus datos con configuraciones avanzadas de seguridad

Principio fundamental: Cuando compartes datos, la seguridad es crítica. Power BI ofrece múltiples capas de protección para asegurar que solo las personas correctas vean la información correcta.

Características de Seguridad

Permisos de Compartición

Controla qué pueden hacer los usuarios con tu informe:

  • Ver: Solo lectura, no puede editar ni compartir
  • Crear contenido: Puede hacer copias y crear informes basados en el dataset
  • Compartir nuevamente: Puede compartir con otros usuarios
Row-Level Security (RLS)

Filtra datos a nivel de fila según el usuario que accede:

  • Define roles con filtros DAX
  • Asigna usuarios a roles específicos
  • Cada usuario ve solo "sus" datos
  • Ejemplo: Gerentes solo ven su región
Roles de Workspace

Permisos jerárquicos en espacios de trabajo:

  • Admin: Control total, gestión de miembros
  • Member: Edita y publica contenido
  • Contributor: Crea contenido, no puede publicar
  • Viewer: Solo lectura del contenido
Cifrado de Datos

Power BI protege datos en múltiples niveles:

  • En tránsito: HTTPS/TLS para transmisión
  • En reposo: Cifrado AES-256 en servidores
  • Autenticación: Azure Active Directory (AAD)
  • Certificaciones: ISO 27001, SOC 1/2

Implementar Row-Level Security (RLS)

Paso a Paso: Configurar RLS
1. En Power BI Desktop - Crear Roles
  1. Modelado → Administrar roles
  2. Crear → Nombra el rol (ej: "Sales_East")
  3. Selecciona tabla a filtrar (ej: Orders)
  4. Agrega filtro DAX:
// Solo mostrar datos de región Este [Region] = "East" // Filtro dinámico por email del usuario [Email] = USERPRINCIPALNAME()
2. Probar Roles Localmente
  1. Modelado → Ver como roles
  2. Selecciona rol a probar
  3. Verifica que datos se filtren correctamente
3. Publicar y Asignar Usuarios
  1. Publica informe a Power BI Service
  2. Ve al Dataset → Seguridad
  3. Selecciona rol
  4. Agrega miembros (usuarios o grupos AAD)
  5. Guardar
Ejemplo práctico: Una empresa con vendedores por región. Cada vendedor solo debe ver ventas de su región. Creas roles "Sales_East", "Sales_West", etc., con filtro [Region] = "East", y asignas usuarios según su región.
Mejores Prácticas de Seguridad
  • Principio de mínimo privilegio: Otorga solo los permisos necesarios
  • Revisa accesos periódicamente: Audita quién tiene acceso a qué
  • No uses "Publicar en web" con datos sensibles (es público sin autenticación)
  • Usa grupos de seguridad AAD: Facilita gestión de usuarios masiva
  • Documenta roles RLS: Mantén registro de qué rol ve qué datos
  • Prueba RLS exhaustivamente: Valida filtros antes de producción
  • Capacita usuarios: Educar sobre manejo responsable de datos

Actividad Práctica Guiada

Aplica DAX y compartición en un ejercicio completo

Ejercicio: Dashboard con DAX y Compartición Segura

Dataset: Usa el Sample_Superstore_Complete.xlsx de la Unidad 3
Parte 1: Crear Medidas DAX (30 min)
  1. Medidas Básicas:
    Total Sales = SUM(Orders[Sales]) Total Profit = SUM(Orders[Profit]) Total Orders = COUNTROWS(Orders) Avg Order Value = DIVIDE([Total Sales], [Total Orders])
  2. Medida de Margen:
    Profit Margin % = DIVIDE( [Total Profit], [Total Sales], 0 // Retorna 0 si división por cero )
  3. Ventas por Categoría (con CALCULATE):
    Furniture Sales = CALCULATE( [Total Sales], Orders[Category] = "Furniture" ) Technology Sales = CALCULATE( [Total Sales], Orders[Category] = "Technology" )
  4. Tabla de Calendario (para Time Intelligence):
    Calendar = CALENDAR( DATE(2014, 1, 1), DATE(2017, 12, 31) )

    Luego relaciona Calendar[Date] con Orders[Order Date]

  5. Medidas YTD (Year To Date):
    Sales YTD = TOTALYTD( [Total Sales], Calendar[Date] ) YTD vs LY = DIVIDE( [Sales YTD] - [Sales YTD LY], [Sales YTD LY] ) Sales YTD LY = CALCULATE( [Sales YTD], SAMEPERIODLASTYEAR(Calendar[Date]) )
Parte 2: Crear Visualizaciones (20 min)
  1. Crea tarjetas para: Total Sales, Total Profit, Profit Margin %, Avg Order Value
  2. Gráfico de barras: Sales por Category (usa medida Total Sales)
  3. Gráfico de líneas: Sales YTD vs Sales YTD LY (muestra comparación temporal)
  4. Tabla: Top 10 productos por Profit (ordena descendente)
  5. Agrega segmentadores para: Year, Category, Region
Parte 3: Configurar RLS (15 min)
  1. Crear Roles por Región:
    • Modelado → Administrar roles
    • Crear rol "East_Region"
    • En tabla Orders, filtro DAX: [Region] = "East"
    • Repetir para West, Central, South
  2. Probar RLS:
    • Modelado → Ver como roles → Selecciona "East_Region"
    • Verifica que solo ves datos de región Este
Parte 4: Publicar y Compartir (15 min)
  1. Guarda el archivo .pbix
  2. Publicar → Selecciona "Mi área de trabajo"
  3. En Power BI Service:
    • Ve al dataset publicado → Seguridad
    • Asigna tu email a rol "East_Region"
    • Guarda
  4. Compartir el informe:
    • Abre el informe
    • Compartir → Ingresa email de un colega (o tu email secundario)
    • Marca "Permitir que los destinatarios compartan"
    • Enviar
  5. Crea una suscripción por email (diaria o semanal)
¡Excelente trabajo! Has creado un dashboard completo con cálculos DAX avanzados, implementado seguridad a nivel de fila y compartido tu informe de forma segura. Estas habilidades son fundamentales en proyectos reales de Power BI.
Desafío adicional: Crea una medida que calcule el ranking de productos por ventas usando RANKX(), y agrega un gráfico que muestre solo el Top 5.
